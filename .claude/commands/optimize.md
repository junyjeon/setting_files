# /optimize - 코드 최적화 및 성능 개선

제공된 코드의 성능을 최적화하고 개선점을 제안해.

## 사용법
```
/optimize [code or file path]
```

## 최적화 영역

### React 컴포넌트
- 불필요한 리렌더링 방지 (React.memo, useMemo, useCallback)
- 번들 크기 최적화 (dynamic imports, code splitting)
- 상태 관리 최적화

### Electron 프로세스
- IPC 통신 최소화
- 메모리 누수 체크
- 프로세스 간 데이터 전송 최적화

### 데이터베이스
- Drizzle 쿼리 최적화
- 인덱스 활용 제안
- N+1 문제 해결

### TypeScript
- 타입 추론 개선
- 제네릭 활용도 향상
- 유니온 타입 최적화

## 분석 항목
1. **현재 성능 이슈 식별**
2. **개선 방안 제시**
3. **예상 성능 향상 수치**
4. **구현 우선순위**

## 출력 형식
```typescript
// 🔍 성능 이슈: [문제 설명]
// ⚡ 개선 방안: [해결 방법]
// 📊 예상 효과: [성능 향상 수치]

// Before
[기존 코드]

// After (최적화됨)
[개선된 코드]
```
